<details open>
<summary><b>рҹҡЁ DRAS 2.0 вҖ” Emergency Management & Response Gateway</b></summary>

<br/>

> A live, deployed system that computes optimal rescue routes across 7 UP cities during multi-incident emergencies вҖ” built as a full DSA showcase.

**The problem:** When disaster strikes, emergency responders need the fastest route вҖ” not just the shortest. They also need to know which hospitals might become isolated (articulation points), where hazards are highest, and how to visit every relief centre efficiently.

**What it does:**
- Plots hospitals, relief centres, and live disaster risk zones (earthquake, storm, nuclear, rail accident, and more) on an interactive map
- Computes rescue routes via **Dijkstra** (shortest distance), **A\*** (fastest with heuristic), and **Min-Max Hazard** (safest path) вҖ” side-by-side
- Identifies **critical infrastructure nodes** using Tarjan's algorithm and **MST** via Kruskal + Union-Find
- Approximates the optimal hospital tour using a **nearest-neighbour + 2-opt TSP** solver
- Shows execution time, nodes visited, and complexity class for every route вҖ” because it's also a DSA explainer
- Supports **live GPS location**, IP fallback, email alerts, and shows all incidents within 20km of you

**Stack:** `Python 3.10` В· `Flask` В· `C++ (engine)` В· `Leaflet.js` В· `OpenStreetMap` В· `Render`

<summary><b>рҹ“Ӣ Attendify 2.0 вҖ” Cloud-Based Attendance Management System</b></summary>

<br/>

> Eliminates proxy attendance and saves 10вҖ“15 minutes of lecture time per class by replacing paper registers with a live digital system.

**The problem:** Traditional roll-calls waste instructional time, and paper registers are trivially exploited вҖ” students mark absent friends present ("proxy"), creating fraudulent records with no accountability.

**What it does:**
- Role-based dashboards for **Admins**, **Teachers**, and **Students** вҖ” each sees exactly what they need
- Teachers mark an entire class in seconds via digital checklist; every entry is timestamped instantly
- Students view their **live attendance percentage** and total-vs-attended stats in real time
- **MongoDB Atlas** NoSQL backend maps Student IDs to Subject IDs as flexible JSON documents
- Deployed on **Render** with CI/CD вҖ” every GitHub push goes live automatically
- Sessions prevent students from accessing teacher or admin routes; env vars keep credentials safe

**Stack:** `Python` В· `Flask` В· `MongoDB Atlas` В· `Gunicorn` В· `Vanilla JS` В· `Render`
